# 네 지역 교류 마을 재현 명세서

이 문서는 같은 성격의 게임을 거의 비슷하게 만들기 위한 기획, 디자인, UI, 조작, 게임 규칙 명세다. 구현 기술은 React와 Phaser를 이용한다는 점만 전제로 한다.

## 1. 게임 개요

게임 제목은 `네 지역 교류 마을`이다. 플레이어는 네 지역 중 하나를 선택해 자기 지역을 성장시키고, 다른 지역과 자원을 교류해 최종 건물을 완성한다.

게임의 핵심 학습 주제는 지역마다 자연환경과 산업이 달라 생산 자원이 다르고, 필요한 것을 얻기 위해 지역 간 교류가 필요하다는 점이다.

전체 분위기는 초등학생이 쉽게 이해할 수 있는 따뜻한 마을 발전 게임이다. 교육용 설명 화면처럼 보이기보다 실제 작은 픽셀 게임을 조작하는 느낌을 목표로 한다.

## 2. 기술 방향

화면 UI는 React를 이용하고, 지도와 캐릭터, 건물, 배치, 이동 애니메이션은 Phaser를 이용한다.

2D 탑다운 시점이며, 픽셀 아트가 선명하게 보이도록 픽셀 렌더링 느낌을 유지한다.

## 3. 전체 화면 흐름

게임은 다음 순서로 진행된다.

1. 시작 화면에서 플레이할 지역을 선택한다.
2. 선택한 지역의 지도 화면으로 들어간다.
3. 자동 생산으로 기본 자원을 모은다.
4. 자원이 충분하면 건물을 단계별로 짓는다.
5. 부족한 자원은 상인을 보내 다른 지역과 교류해 얻는다.
6. 제작 건물을 지으면 지역 생산품을 만든다.
7. 6단계 최종 건물을 완성하면 성공 화면을 보여준다.

현재 게임은 한 명이 하나의 지역을 선택해 진행하는 방식이다. 화면에는 선택한 지역이 크게 표시되고, 다른 지역은 교류 대상 개념으로 등장한다.

## 4. 지역 구성

지역은 총 4개다.

| 지역 | 기본 자원 | 생산품 | 역할 분위기 |
| --- | --- | --- | --- |
| 산간 지역 | 나무 | 목재 상자 | 숲, 나무, 목공, 산림 마을 |
| 공업 지역 | 부품 | 수레 | 공장, 조립, 기계, 물류 |
| 농촌 지역 | 곡식 | 도시락 | 밭, 창고, 방앗간, 농장 |
| 해안 지역 | 해산물 | 해산물 꾸러미 | 바다, 항구, 어장, 시장 |

각 지역은 자기 기본 자원 3개를 가지고 시작한다. 다른 기본 자원과 모든 생산품은 0개에서 시작한다. 일꾼은 1명으로 시작하고, 상인은 처음에 없다.

## 6. 핵심 게임 규칙

자동 생산은 10초마다 발생한다. 생산량은 기본적으로 현재 일꾼 수만큼이다. 자동 생산이 강화되면 추가 생산량이 붙는다.

예를 들어 일꾼 1명, 자동 생산 보너스 0이면 10초마다 기본 자원 1개를 얻는다. 일꾼 2명, 자동 생산 보너스 1이면 10초마다 기본 자원 3개를 얻는다.

농촌에서 강아지를 입양하면 자동 생산 시 일정 확률로 곡식을 1개 더 얻는다. 닭을 데려오면 자동 생산 때 달걀이 1개씩 늘어난다.

건물을 완성하면 지역 발전도가 오른다. 초반 건물은 소폭, 중후반 건물은 더 크게 오른다. 제작과 교류도 발전도를 올린다.

## 7. 건설 방식

건설은 하단의 `건설하기` 버튼으로 시작한다.

건설 창에는 현재 지역의 6개 건물이 카드 형태로 표시된다. 각 카드는 건물 그림, 이름, 상태, 효과, 필요 자원을 보여준다.

건물 상태는 다음 네 가지로 구분한다.

| 상태 | 의미 |
| --- | --- |
| 완성 | 이미 지은 건물 |
| 건설 가능 | 현재 단계이고 자원이 충분함 |
| 자원 부족 | 현재 단계이지만 자원이 부족함 |
| 잠김 | 이전 단계가 완성되지 않아 아직 지을 수 없음 |

건설 가능한 건물을 선택하고 `건설`을 누르면 지도 배치 모드가 된다. 배치 모드에서는 지도 위에 반투명 건물 미리보기가 따라다닌다.

플레이어는 자기 지역 안의 원하는 위치를 탭하거나 클릭해 건물을 놓는다. 이미 지어진 건물과 너무 가까운 곳에는 배치할 수 없다. 배치 가능 위치는 초록색 느낌의 표시, 불가능 위치는 붉은색 느낌의 표시로 보여준다.

건물이 놓이면 필요한 자원이 차감되고, 지도 위에 실제 건물 그래픽이 생긴다. 완료 알림은 `완성!`처럼 짧게 표시한다.

## 8. 건물 상호작용

지도 위의 건물을 클릭하면 건물 상태 패널이 열린다.

상태 패널에는 다음 정보가 표시된다.

| 항목 | 내용 |
| --- | --- |
| 지역 | 건물이 속한 지역 |
| 건물 이름 | 예: 작은 밭, 방앗간 |
| 건물 이미지 | 해당 단계 건물 그림 |
| 단계 | 1~6 |
| 역할 | 생산, 창고, 쉼터, 제작, 자동 생산, 최종 완성 |
| 상태 | 현재 가능한 행동 또는 완료 상태 |

건물 단계에 따라 버튼이 달라진다.

| 건물 | 가능한 행동 |
| --- | --- |
| 1단계 | 상인 뽑기 |
| 농촌 2단계 | 강아지 입양 |
| 농촌 3단계 | 닭 데려오기, 달걀 모으기, 달걀 선물하기 |
| 4단계 제작 건물 | 생산품 만들기, 특별 생산품 만들기 |
| 5단계 고급 건물 | 자동 생산 강화 |
| 6단계 최종 건물 | 완성 확인 |

상인은 1단계 건물마다 1명만 배치할 수 있다. 이미 상인이 배치된 건물은 다시 뽑을 수 없다.

## 9. 교류 방식

교류는 하단의 `교류하기` 버튼으로 시작한다. 교류하려면 먼저 1단계 건물을 클릭해서 상인을 뽑아야 한다.

교류 창에는 상인 목록이 표시된다. 대기 중인 상인만 선택할 수 있고, 이동 중인 상인은 비활성화된다.

교류 대상은 자기 지역을 제외한 나머지 지역 중 하나를 고른다. 교류 비율은 고정이다.

내 기본 자원 2개 -> 상대 지역 기본 자원 1개

수량은 2개 단위로 늘리거나 줄인다. 예를 들어 곡식 2개를 보내면 나무 1개를 받고, 곡식 4개를 보내면 나무 2개를 받는다.

상인을 보내면 내 자원이 즉시 차감되고, 상인은 지도 위에서 이동 중 상태가 된다. 약 4.8초 뒤 상인이 돌아오며 받을 자원이 추가된다. 교류 완료 시 발전도도 오른다.

교류의 목적은 자기 지역 자원만으로는 중후반 건물을 완성할 수 없게 만들어, 다른 지역과의 상호 의존성을 경험하게 하는 것이다.

## 10. 제작 방식

4단계 제작 건물을 지은 뒤 해당 건물을 클릭하면 생산품을 만들 수 있다.

제작 버튼을 누르면 레시피에 맞는 기본 자원이 차감되고 생산품 1개가 추가된다. 제작에 성공하면 발전도가 오른다.

특별 생산품 만들기는 기본 레시피보다 자기 지역 기본 자원 1개를 더 요구한다. 결과물은 같은 생산품 1개지만 발전도 보상이 조금 더 크다.

생산품은 최종 건물을 짓기 위한 핵심 재료다. 각 지역의 최종 건물에는 자기 지역 생산품 2개가 필요하다.

## 11. 성공 조건

선택한 지역의 6단계 최종 건물을 완성하면 성공 화면이 열린다.

성공 화면은 다음 내용을 보여준다.

| 항목 | 내용 |
| --- | --- |
| 완료 지역 | 선택한 지역 이름 |
| 교환 성공 횟수 | 상인 교류 완료 횟수 |
| 선물 횟수 | 달걀 선물 등 선물 행동 횟수 |
| 생산품 제작 수 | 제작 성공 횟수 |
| 최종 건물 | 1/1 |
| 지역 발전도 | 최종 발전도 |
| 지역별 결과 | 네 지역의 발전도 요약 |

성공 메시지는 “자기 지역의 자원만으로는 끝까지 발전할 수 없었고, 다른 지역과 필요한 자원을 주고받아 최종 건물을 완성했다”는 내용을 담는다.

## 12. 조작 방식

마우스와 터치를 모두 지원한다.

| 조작 | 동작 |
| --- | --- |
| 시작 화면 카드 클릭 | 플레이 지역 선택 |
| 지도 드래그 | 카메라 이동 |
| 건물 클릭 | 건물 상태 패널 열기 |
| 건설하기 버튼 | 건설 창 열기 |
| 건설 창의 건설 버튼 | 배치 모드 진입 |
| 배치 모드에서 지도 클릭 | 건물 배치 |
| 배치 모드 취소 버튼 | 배치 취소 |
| 교류하기 버튼 | 교류 창 열기 |
| 상인 카드 클릭 | 교류에 사용할 상인 선택 |
| 지역 버튼 클릭 | 교류 대상 선택 |
| 아래/위 버튼 | 보낼 자원 수량 조절 |
| 보내기 버튼 | 상인 출발 |

지도 드래그와 클릭은 구분되어야 한다. 조금 움직인 경우는 드래그로 보고, 거의 움직이지 않은 경우만 탭 또는 클릭으로 처리한다.

배치 모드에서는 지도 드래그보다 건물 위치 선택이 우선이다. 플레이어가 커서나 손가락을 움직이면 미리보기 위치가 바뀌고, 탭하면 해당 위치에 배치한다.

## 13. 지도와 카메라

지도는 선택한 지역 하나를 크게 보여주는 구조다. 지역 배경 이미지는 화면보다 크게 잡아, 플레이어가 드래그로 내부를 둘러볼 수 있게 한다.

카메라는 지역 중앙을 기본 위치로 잡는다. 화면 크기가 바뀌면 지도와 카메라 범위도 다시 맞춘다.

지도 위에는 다음 요소가 표시된다.

| 요소 | 표시 방식 |
| --- | --- |
| 지역 배경 | 전체 화면을 채우는 픽셀 아트 배경 |
| 일꾼 | 지역 안을 걷거나 작업하는 캐릭터 |
| 건물 | 플레이어가 배치한 위치에 표시 |
| 상인 | 대기 중이면 중앙 근처, 이동 중이면 왕복 이동 |
| 동물 | 농촌에서 강아지와 닭을 입양하면 표시 |
| 배치 미리보기 | 건설 중인 건물의 반투명 표시 |
| 플로팅 텍스트 | 자원 획득, 완성, 교류 완료 등 |

## 14. 비주얼 스타일

전체 아트는 귀여운 소프트 픽셀 스타일이다. 너무 낡은 레트로 느낌보다는 밝고 선명한 현대식 픽셀 게임 느낌을 목표로 한다.

핵심 키워드는 다음과 같다.

| 항목 | 방향 |
| --- | --- |
| 시점 | 2D 탑다운 |
| 분위기 | 따뜻함, 귀여움, 협력, 성장 |
| 색감 | 크림, 나무색, 자연색, 지역별 포인트색 |
| UI 재질 | 나무판, 종이, 게시판, 설계도 |
| 선 처리 | 진한 갈색 테두리 |
| 버튼 | 두껍고 눌리는 게임 버튼 느낌 |
| 텍스트 | 짧고 굵게, 초등학생이 읽기 쉽게 |

피해야 할 요소는 유리처럼 투명한 UI, 과도한 그라데이션, 미래형 UI, 광고 게임 같은 과장된 보상 연출, 일반 업무용 웹앱 같은 차가운 화면이다.

## 16. UI 레이아웃

게임 화면은 맵이 주인공이다. UI는 맵 위에 얹히지만, 맵을 과하게 가리지 않아야 한다.

기본 배치는 다음과 같다.

| 위치 | 요소 |
| --- | --- |
| 좌상단 | 내 지역 HUD |
| 중앙 전체 | 지도 |
| 하단 중앙 | 주요 행동 버튼 2개 |
| 하단 중앙 위 | 알림판 |
| 우측 또는 중앙 우측 | 건물 상태 패널 |
| 중앙 | 모달 창 |

좌상단 HUD에는 자원, 생산품, 일꾼 수, 상인 수, 발전도, 자동 생산 남은 시간이 표시된다.

하단 액션바에는 `건설하기`, `교류하기` 두 버튼이 있다. 버튼은 나무판 위에 놓인 큼직한 게임 버튼처럼 보여야 한다.

알림판은 화면 하단 중앙에 위치하고, 현재 행동 결과나 안내 문구를 짧게 보여준다.

## 17. 시작 화면

시작 화면은 농촌 지역 배경 이미지를 화면 전체 배경으로 사용한다. 가운데에는 나무/종이 질감의 패널을 놓는다.

표시 문구는 다음 구조를 따른다.

| 요소 | 내용 |
| --- | --- |
| 작은 제목 | 네 지역 교류 마을 |
| 큰 제목 | 플레이할 지역을 고르세요 |
| 설명 | 혼자 하나의 지역을 맡아 자원을 모으고, 다른 지역과 교환하며 최종 건물을 완성합니다. |
| 지역 카드 | 산간, 공업, 농촌, 해안 |

지역 카드에는 지역 이름, 생산 자원, 시작 자원 수량을 보여준다. 카드를 누르면 바로 게임이 시작된다.

## 18. HUD 명세

HUD는 작고 촘촘하지만 읽기 쉬워야 한다.

자원 배지는 다음 항목을 표시한다.

| 배지 | 표시 |
| --- | --- |
| 곡식 | 현재 보유 곡식 수 |
| 해산물 | 현재 보유 해산물 수 |
| 나무 | 현재 보유 나무 수 |
| 부품 | 현재 보유 부품 수 |
| 지역 생산품 | 도시락, 꾸러미, 상자, 수레 중 해당 지역 생산품 |
| 달걀 | 농촌에서 달걀이 1개 이상 있을 때만 표시 |

미니 상태 줄에는 다음 정보를 표시한다.

| 항목 | 예시 |
| --- | --- |
| 일꾼 | 일꾼 1명 |
| 상인 | 상인 1/2명 |
| 발전도 | 발전도 24 |
| 자동 생산 | 자동 생산 7초 |

## 19. 모달 디자인

모달은 종이 설계도나 장터 장부처럼 보이게 만든다. 배경은 반투명 어두운 오버레이로 처리하고, 모달 카드에는 종이 질감과 두꺼운 갈색 테두리를 사용한다.

닫기 버튼은 우측 상단에 둔다. 버튼 문구는 짧게 `닫기`로 표시한다.

건설 모달은 다른 모달보다 넓다. 건물 카드 6개가 여러 열로 배치되고, 작은 화면에서는 한 열로 쌓인다.

교류 모달은 상인 선택, 대상 지역 선택, 수량 조절, 보내기 버튼 순서로 구성한다.

## 20. 알림과 이펙트

일반 알림은 하단 알림판에 표시한다. 예시는 다음과 같다.

| 상황 | 문구 예시 |
| --- | --- |
| 게임 시작 | 농촌 지역을 선택했어요. 자원을 모아 지역을 발전시켜 보세요. |
| 자원 부족 | 자원이 부족합니다. |
| 배치 안내 | 방앗간을 지을 위치를 내 지역 안에서 선택하세요. |
| 배치 불가 | 그 위치에는 지을 수 없어요. 빈 땅을 선택하세요. |
| 건설 완료 | 방앗간 완성! 지역 발전도가 올랐어요. |
| 교류 출발 | 상인1이 곡식 2개를 싣고 산간 지역으로 출발했어요. |
| 교류 완료 | 상인1이 나무 1개를 가져왔어요. |

지도 위 플로팅 텍스트는 짧게 위로 떠오르며 사라진다. 표시 시간은 약 1.2초 정도가 적당하다.

예시는 `+1 곡식`, `+1 달걀`, `완성!`, `배치 불가`, `자동 +1`이다.

## 21. 캐릭터와 애니메이션

캐릭터는 4방향 걷기 애니메이션을 기본으로 한다. 농촌에는 수확 애니메이션이 추가된다.

일꾼은 지역 안에서 자동으로 움직이거나 작업한다. 농촌의 수확 일꾼은 주기적으로 `+곡식` 텍스트를 띄워 살아 있는 마을 느낌을 준다.

상인은 대기 중일 때는 지역 중앙 근처에 서 있고, 교류 중일 때는 짐수레를 끌고 목표 지역 방향으로 이동한 뒤 다시 돌아오는 연출을 한다.

농촌에서 강아지와 닭을 얻으면 지도 위에 동물 그래픽이 나타나고, 작은 경로를 따라 천천히 움직인다.

## 23. 반응형 기준

화면은 태블릿과 노트북에서 안정적으로 보여야 한다.

전체 게임 화면은 100vh 또는 100dvh를 사용해 브라우저 화면을 꽉 채운다. 스크롤이 생기지 않도록 한다.

작은 화면에서는 다음처럼 조정한다.

| 요소 | 조정 |
| --- | --- |
| HUD | 폭을 줄이고 자원 배지를 더 촘촘히 배치 |
| 알림판 | 글자 크기 축소 |
| 건설 카드 | 한 열로 표시 |
| 모달 | 화면 폭에 맞게 줄이고 내부 스크롤 허용 |
| 액션바 | 하단 중앙에 고정, 버튼 2개 유지 |

터치 조작을 고려해 버튼은 너무 작게 만들지 않는다.

## 25. 재현 시 핵심 체크리스트

비슷한 게임으로 보이려면 다음 요소를 반드시 맞춘다.

| 항목 | 필수 여부 |
| --- | --- |
| 4개 지역과 각기 다른 기본 자원 | 필수 |
| 시작 지역 선택 화면 | 필수 |
| 10초 자동 생산 | 필수 |
| 6단계 순차 건설 | 필수 |
| 지도 위 직접 건물 배치 | 필수 |
| 건물 간 배치 거리 제한 | 필수 |
| 1단계 건물에서 상인 모집 | 필수 |
| 자원 2개 -> 다른 지역 자원 1개 교류 | 필수 |
| 교류 중 상인 이동 연출 | 필수 |
| 4단계 제작 건물과 생산품 제작 | 필수 |
| 최종 건물 성공 화면 | 필수 |
| 나무/종이 질감 UI | 필수 |
| 소프트 픽셀 아트 | 필수 |
| 농촌 강아지/닭/달걀 상호작용 | 재현도를 높이려면 권장 |

이 게임의 핵심은 단순히 자원을 모아 건물을 짓는 것이 아니라, “내 지역만으로는 완성할 수 없어서 다른 지역과 교류해야 한다”는 흐름이다. 따라서 건물 비용에는 반드시 다른 지역 자원이 섞여 있어야 하고, 최종 건물에는 생산품까지 요구해야 한다.
